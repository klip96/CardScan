# CardScan Android (Capacitor)

Тонкая обёртка над веб-PWA CardScan в WebView, чтобы получить устанавливаемый
`.apk`. Сервер остаётся на офисном ПК в локальной Wi-Fi сети — приложение не
хранит адрес сервера в сборке, а спрашивает его при первом запуске (см.
`www/index.html`) и сохраняет на телефоне. Изменить адрес позже — очистить
данные приложения или переустановить.

## Что уже готово в этом репозитории

- `package.json`, `capacitor.config.json`, `www/index.html` — сам Capacitor-проект.
- `android/` — сгенерированный Gradle-проект (`npx cap add android`), с:
  - иконками, сгенерированными из `app/web/icon-512.png`;
  - `android:usesCleartextTraffic="true"` в манифесте (сервер офиса — обычный HTTP, не HTTPS);
  - разрешениями `INTERNET` (остальное — стандартный boilerplate Capacitor).
- `android-sdk/` — локально установленный Android SDK (cmdline-tools,
  platform-tools, `platforms;android-34`, `build-tools;34.0.0`). Не коммитится
  в git (см. корневой `.gitignore`) — папка большая (~1–2 ГБ).

## Сборка APK

```powershell
cd android-app
.\build.ps1
```

Готовый файл: `android\app\build\outputs\apk\debug\app-debug.apk` (debug-подпись,
достаточно для установки через «неизвестные источники» — в Play Store
приложение не публикуется).

### Важно: сборка требует ОБЫЧНОГО терминала, не песочницы агента

При сборке этого проекта через Claude Code в этом окружении `gradlew.bat`
падает с ошибкой:

```
java.io.IOException: Unable to establish loopback connection
Caused by: java.net.SocketException: Invalid argument: connect
    at sun.nio.ch.UnixDomainSockets.connect0(Native Method)
```

Это не баг конфигурации проекта — это известное ограничение JDK 17+ на Windows
внутри песочниц вида Microsoft Store/AppContainer (см.
[JDK-8312215](https://bugs.openjdk.org/browse/JDK-8312215)): внутренний
NIO-пайп Gradle-коннектора (`Selector.open()`/`Pipe.open()`) использует
Unix Domain Socket для loopback-соединения, а такие сокеты заблокированы
именно в песочнице, в которой выполняются команды агента — при этом обычные
TCP-сокеты на 127.0.0.1 работают нормально (проверено отдельным тестом).
Подтверждено: **вне** песочницы (обычный PowerShell/cmd, Android Studio) эта
проблема не должна воспроизводиться, так как ограничение накладывает именно
контейнер, в котором работает агент, а не сама система/JDK/проект.

Поэтому: откройте обычный PowerShell (например, через Проводник или меню
Пуск — не через Claude Code) и запустите `.\build.ps1` там.

## Первый запуск на телефоне

1. Установите `app-debug.apk` (нужно разрешить установку из неизвестных источников).
2. Откройте приложение — оно попросит адрес сервера. На ПК откройте
   `http://<ip-пк>:8000/connect` — там показан IP и QR-код.
3. Введите адрес (например `http://192.168.1.50:8000`) — приложение
   перейдёт на страницу входа сервера и дальше работает как обычный CardScan.

## Известный риск, который нужно проверить на реальном устройстве

Съёмка визитки использует `<input type="file" capture="environment">` —
штатный HTML-способ вызвать камеру. В Android WebView (то, что использует
Capacitor) это обычно работает, но версии Android/WebView отличаются.
**Проверьте съёмку камерой на реальном телефоне сразу после установки.**
Если не сработает — есть официальный плагин `@capacitor/camera` (нативный
вызов камеры) как запасной вариант; сейчас он не подключён, чтобы не менять
JS без необходимости.
